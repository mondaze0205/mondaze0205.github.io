# 🚀 리액트(React)와 친해지기

이 포트폴리오 프로젝트를 직접 만들면서 배운 리액트의 핵심 개념들을 나중에 내가 다시 봐도 이해할 수 있게 정리해 보았다.

## 1. 컴포넌트(Component) - "조립식 장난감"
리액트에서 모든 것은 '컴포넌트'다. 화면을 구성하는 하나하나의 블록이라고 생각하면 쉽다.
- **Window:** 앱을 담는 창 틀
- **PaintApp:** 내가 만든 그림판 내부
- **Dock:** 하단의 아이콘 바
이렇게 나눠서 만드니까 나중에 고치기도 편하고, 필요한 곳에 다시 가져다 쓰기도 좋다!

## 2. 상태(State) - "앱의 기억력"
`useState`는 앱이 무언가를 기억하게 할 때 사용한다.
- **창이 열려 있나?** (openWindows)
- **그림판에서 지금 무슨 색깔을 쓰고 있나?** (color)
- **음악이 재생 중인가?** (isPlaying)
상태값이 바뀌면 리액트가 알아서 화면을 다시 그려준다(렌더링). 이게 리액트의 제일 편한 점이다!

## 3. 훅(Hooks) - "리액트의 특수 도구"
프로젝트에서 정말 많이 쓴 '마법 도구' 3가지:

### ✨ useEffect (자동 실행기)
- 음악 앱을 열자마자 유튜브 API를 불러오거나, 캔버스를 처음 만들 때 사용했다.
- "화면이 처음 나타날 때(Mount) 딱 한 번만 이거 해줘!"라고 시킬 때 아주 유용하다.

### 📍 useRef (직접 가리키기)
- 특히 **그림판(Canvas)** 만들 때 핵심이었다!
- HTML의 `<canvas>` 태그를 직접 만져서 그림을 그려야 할 때, `ref`라는 깃발을 꽂아서 리액트가 그 태그를 기억하게 한다.

### 🔄 useCallback (기능 기억하기)
- 그림판의 **Undo(되돌리기)** 기능을 만들 때 썼다.
- 매번 새로 함수를 만드는 대신, 한 번 만든 기능을 기억해뒀다가 재사용해서 성능을 높여준다.

## 4. 프롭스(Props) - "데이터 배달"
컴포넌트끼리 데이터를 주고받을 때 사용한다.
- `Window` 컴포넌트에게 `title="Paint"`라고 알려주면, 창 제목에 "Paint"라고 적히는 원리다. 부모가 자식에게 주는 선물 같은 느낌!
